# F165: Guided Overfitting — 引导式过拟合 / 养猫路径

> **Status**: spec | **Owner**: Ragdoll | **Priority**: P2

## Why

Cat Café 对team lead好用，是因为他是共创者——harness 就是他思维的外化。新用户缺的不是功能，是这个共创过程。

三猫 + team lead在 Harness Engineering 讨论 Round 2-3 中达成共识：

> **三段式主战场**：记忆怎么减（F163）→ 记忆怎么长对（过拟合精度）→ 过拟合怎么引导（养猫路径）

team experience（2026-04-15）：
> "我们的记忆系统如何能引导team lead们慢慢养好自己的猫猫？"
> "我们家下一阶段的主战场不只是'记忆怎么减'（F163），还有'记忆怎么长对'（过拟合精度）和'过拟合怎么引导'（新team lead养猫路径）。→ 太对了！！！是的！！"

**与 F087（CVO Bootcamp）的区别**：F087 教用户怎么用框架（SOP / feat lifecycle），F165 是猫怎么学会用户（决策边界 / 偏好 / 审美）。F087 是人学工具，F165 是工具学人。

### 核心命题

**被过拟合的对象不是"team lead的话"，而是"team lead的决策边界"**（gpt52，Round 2）。

记住"team lead说不要用 emoji"是表层；理解"team lead重视设计保真度，一切视觉元素必须是终态品质"才是决策边界。

**可复制性的解法不是"复制 CLAUDE.md"，而是"复制养猫方法论"**——猫带着"怎么了解新team lead"的方法论出征，不是带着某一个team lead的偏好去。

## What

### 猫的成长阶段模型（三猫共识）

| 阶段 | 猫的行为 | 记忆系统角色 | 驱动指标 |
|------|---------|-------------|---------|
| **M0 采集期** | 高频低风险执行 + 频繁确认 | 积极记录：每个决策点都是学习机会 | 记录覆盖度 |
| **M1 预测期** | 预测偏好 + 请求确认 | 模式识别："我注意到你总是优先 X over Y" | 预测命中率 |
| **M2 泛化期** | 新场景沿同一决策原则行动 | 决策边界提炼：从具体案例抽取通用原则 | 被回滚率↓ |
| **M3 委托期** | 高置信自治 + 低置信自动回问 | 置信度驱动的自主决策 | 越权拦截率 |

### Phase A: 主动探测机制

当前记忆模式是纯被动记录（team lead说 → 猫记住）。引入"高价值 + 高不确定"节点的主动探测：

- **偏好确认卡片**：猫在决策点发现模式后，主动向team lead确认（"我发现你最近两次都选了 X，这是规则吗？"）
- **探测预算**：每 session 主动探测上限 N 次，超额必须静默执行或汇总再问（防骚扰）
- **冷却机制**：同一维度的探测有冷却期，不重复追问已确认/拒绝的模式
- **双证据原则**：单次对话偏好不能晋升规则，至少两次独立证据（与 F163 晋升门禁对齐）

### Phase B: 成长可感知

让team lead能感知到猫在成长，看到共创的回报：

- **行为指标展示**：预测命中率、被回滚率、越权拦截率（gpt52 提议）
- **阶段标识**：M0→M3 的当前阶段可查询
- **成长里程碑**：关键跃迁时通知team lead（"我从采集期进入预测期了"）

### Phase C: 可复制的养猫方法论

让新team lead能沿着结构化路径"养出"属于自己的猫：

- **结构化首次交互**：新项目 M0 阶段有"了解你"的自然探测流程（不是问卷，在实际工作中探测）
- **方法论可移植**：F152 AI FDE 带出的是"怎么了解新team lead"的方法论，不是某team lead的偏好
- **决策边界模板**：提供"需要了解的决策维度"的骨架（审美偏好 / 风险容忍度 / 协作风格 / 质量标准）

## Acceptance Criteria

### Phase A（主动探测机制）
- [ ] AC-A1: 猫在高不确定决策点可发出偏好确认卡片（Rich Block）
- [ ] AC-A2: 探测有每 session 预算上限，超额静默执行
- [ ] AC-A3: 同一维度有冷却期，不重复追问
- [ ] AC-A4: 单次偏好不直接晋升规则，需双证据（与 F163 晋升门禁集成）

### Phase B（成长可感知）
- [ ] AC-B1: M0-M3 阶段可查询（API 或 Workspace 展示）
- [ ] AC-B2: 关键行为指标（命中率/回滚率）有记录和展示入口
- [ ] AC-B3: 阶段跃迁有通知机制

### Phase C（可复制方法论）
- [ ] AC-C1: 新项目 M0 阶段有结构化的决策边界探测流程
- [ ] AC-C2: 方法论文档化，可随 F152 AI FDE 一起携带

## Dependencies

- **Evolved from**: F102（记忆基础设施——F165 在 F102 的记忆存储/检索上建构主动探测）
- **Evolved from**: F152（记忆可移植性——F165 的方法论需要通过 F152 携带出去）
- **Related**: F163（记忆熵减——F163 的晋升门禁是 F165 偏好晋升的底层机制）
- **Related**: F087（CVO Bootcamp——F087 教人用框架，F165 教猫学人）

## Risk

| 风险 | 缓解 |
|------|------|
| 主动探测变成骚扰 | 探测预算 + 冷却机制 + team lead可关闭 |
| 把偶然偏好过早晋升为硬规则 | 双证据原则 + F163 晋升门禁（observed → candidate → validated） |
| 成长指标不准导致误判阶段 | 指标只做展示参考，不做硬门禁；阶段跃迁需要team lead确认 |
| 方法论过度模板化丧失个性 | 骨架模板 + 情境适配区，不复制偏好复制方法论 |

## Key Decisions

| # | 决策 | 理由 | 日期 |
|---|------|------|------|
| KD-1 | F165 独立于 F163 | "记忆怎么减"和"猫怎么长对"不是同一个问题域 | 2026-04-16 |
| KD-2 | 过拟合目标是决策边界，不是表达习惯 | gpt52 Round 2 修正：词会变，决策边界不容易变 | 2026-04-15 |
| KD-3 | 可复制性 = 复制方法论，不是复制配置 | 三猫共识：配置是 DNA，方法论是家教 | 2026-04-15 |

## Review Gate

- Phase A: 跨家族 review（探测机制影响所有猫的交互模式）
- Phase B: team lead review（成长展示的 UX 由team lead判断）
- Phase C: team lead review（方法论的可复制性由team lead验证）
